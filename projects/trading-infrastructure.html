Key Design Decisions

ğŸ’¡ Why Ephemeral Containers?

Compute containers (data collector, ML engine) run with --rm flag and are destroyed after each pipeline run. This ensures clean state, prevents resource leaks, and allows broker connections from multiple locations.

ğŸ’¡ Why Self-hosted Runner?

GitHub-hosted runners cannot access the private network where broker connectivity is established. Self-hosted runner on the VPS enables direct Docker control and local network access.

ğŸ’¡ Why Separate Networks?

Trading and monitoring services are isolated in separate Docker networks. This provides security boundaries and prevents monitoring stack issues from affecting trading operations.

ğŸ’¡ Why Thread-safe Model Loading?

ML models (~1GB) are loaded once with mutex locks to prevent race conditions and OOM errors when processing multiple tickers concurrently with ThreadPoolExecutor.
